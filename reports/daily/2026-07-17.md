# Agent 技术日报 · 2026-07-17

## GitHub projects

### [awesome-llm-apps](https://github.com/Shubhamsaboo/awesome-llm-apps)
- 包含100+个可实际运行的AI Agent和RAG应用，支持MCP与tool calling集成。
- 通过multi-agent协作框架实现复杂任务编排，克隆即用可自定义。
- 覆盖多种场景的RAG流水线，结合工具调用与Agent协作快速部署。

### [cc-switch](https://github.com/farion1231/cc-switch)
- cc-switch 是一款跨平台桌面助手，整合了 MCP（Model Context Protocol）服务，通过标准化接口连接多种 AI 模型与外部工具，实现高效的 tool calling 能力。
- 它内置 RAG（检索增强生成）功能，支持本地知识库检索，提高 Claude Code、Codex、OpenCode 等 Agent 回答的准确性与上下文相关性。
- 通过 multi-agent 架构，cc-switch 可协调 Claude Code、Codex、OpenCode、OpenClaw、Gemini CLI 和 Hermes Agent 等多个智能体协同工作，完成复杂任务。

### [gemini-cli](https://github.com/google-gemini/gemini-cli)
- 支持MCP协议，通过命令行与终端交互，将Gemini模型作为AI agent运行。
- 内置RAG能力，可检索本地文件或知识库，辅助上下文理解与决策。
- 集成tool calling与multi-agent架构，允许动态调用外部工具并协调多个智能体协作。

### [browser-use](https://github.com/browser-use/browser-use)
- 通过MCP协议让AI智能体像人类一样操作浏览器，实现RAG数据采集和网页自动化交互。
- 结合tool calling动态调用浏览器功能，完成表单填写、页面跳转等复杂任务。
- 支持multi-agent协作，多个智能体协同完成跨网站数据整合与流程串联。

### [ponytail: 让AI代理像最懒的高级开发者一样思考](https://github.com/DietrichGebert/ponytail)
- 通过MCP协议集成，自动将复杂任务拆解为最小可行步骤，优先调用现有工具避免重复编码
- 结合RAG从历史代码库中检索最佳实践，拒绝重写已存在的解决方案
- 基于multi-agent架构，让代理在tool calling时自动协商分工，确保每个子任务都用最少的代码实现

### [open-design](https://github.com/nexu-io/open-design)
- 🎨 通过 MCP 集成 AI 设计代理，支持 Claude、Gemini 等模型，实现自然语言驱动的 UI 生成与迭代。
- 📝 利用 RAG 技术从设计文档和组件库中检索上下文，提升生成质量；支持 tool calling 执行图像处理、文件转换等操作。
- 🤖 基于 multi-agent 架构协调代码生成、布局优化与导出任务，最终产出 HTML、PDF、PPTX、MP4 等真实文件。
