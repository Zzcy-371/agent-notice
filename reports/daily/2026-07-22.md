# Agent 技术日报 · 2026-07-22

## GitHub projects

### [pi: AI Agent Toolkit](https://github.com/earendil-works/pi)
- 提供统一LLM API接口和工具调用（Tool Calling）能力，支持MCP（Model Context Protocol）集成与RAG（检索增强生成）知识库。
- 内置多智能体（Multi-Agent）编排框架，支持复杂任务分解与协作。
- 包含终端UI（TUI）和编码代理CLI，可交互式运行Agent循环。


## Research and engineering

### [A simple API for offering your coding agent a smoke break](https://smoke-break.pineapplefreefall.com)
- 为编码代理提供休息API，集成MCP和RAG工具调用，优化agent工作流。
- 通过tool calling实现多代理（multi-agent）协作，支持动态任务分配。
- 利用RAG增强代理知识库，结合MCP协议提升上下文管理效率。

### [How to Run a Multi-Agent Coding Workspace (2026)](https://www.augmentcode.com/guides/how-to-run-a-multi-agent-coding-workspace)
- Orchestrate multiple LLM agents with distinct roles (e.g., coder, reviewer) via MCP for coordinated code generation and review.
- Integrate project context and documentation using RAG to provide agents with up-to-date knowledge during development.
- Leverage tool calling (e.g., git, linters, test runners) and inter-agent communication to automate complex multi-agent workflows.

### [OpenBench – A benchmark for comparing coding-agent harnesses](https://twitter.com/mattlam_/status/2079605387121049605)
- OpenBench 提供一个标准化基准，用于评估和比较不同的编程智能体框架，涵盖 MCP、RAG 和 tool calling 等关键能力。
- 该基准通过 multi-agent 协作任务测试各框架在复杂编码场景下的表现，包括代码生成、调试与工具调用。
- OpenBench 旨在推动 coding-agent 框架的透明对比，帮助开发者选择最优的 RAG 检索策略和多智能体协调方案。
