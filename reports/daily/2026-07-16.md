# Agent 技术日报 · 2026-07-16

## GitHub projects

### [Superpowers：一种有效的智能体技能框架与软件开发方法论](https://github.com/obra/superpowers)
- 结合MCP与RAG，构建可扩展的智能体技能库，实现高效的知识检索与工具调用。
- 通过tool calling机制，让智能体动态调用外部API与本地函数，完成复杂任务编排。
- 支持multi-agent协作架构，多个专业智能体协同工作，提升软件开发与运维效率。

### [ECC: Agent Harness Performance Optimization System](https://github.com/affaan-m/ECC)
- 集成了MCP（模型上下文协议）、RAG（检索增强生成）和tool calling（工具调用）能力，为Claude Code、Codex、Opencode、Cursor等平台提供统一的执行环境优化。
- 通过multi-agent（多智能体）协作机制协调技能、直觉、记忆与安全模块，实现高效的任务分配与资源调度。
- 采用research-first（研究优先）的开发范式，持续迭代系统性能，确保在不同Agent框架中保持低延迟与高吞吐量。

### [hermes-agent](https://github.com/NousResearch/hermes-agent)
- 支持 MCP 协议集成，可动态扩展工具集，实现灵活的 tool calling 能力
- 内置 RAG 流水线，能够从私有知识库中检索上下文，增强回答准确性和实时性
- 采用 multi-agent 架构，通过编排多个专业子代理协作完成复杂任务，持续进化

### [opencode](https://github.com/anomalyco/opencode)
- 开源编码代理（open source coding agent），集成 MCP 协议实现标准化工具调用。
- 通过 RAG 技术增强上下文理解，支持多模态知识检索辅助代码生成。
- 支持多智能体协作（multi-agent），结合 tool calling 实现自动化任务编排。

### [skills](https://github.com/anthropics/skills)
- 该仓库提供MCP、RAG、tool calling等技能的实现与示例。
- multi-agent协作系统通过集成tool calling实现复杂任务分配。
- 基于RAG的知识检索增强工具链，支持agent动态调用MCP服务。

### [langflow](https://github.com/langflow-ai/langflow)
- Langflow 支持通过可视化界面构建多个 MCP 服务器，简化 agent 与外部工具和数据源的集成。
- 它内置 RAG 管道和 tool calling 机制，使 agent 能动态检索信息并调用 API 或自定义函数。
- Langflow 允许创建 multi-agent 系统，通过编排不同 agent 协作完成复杂工作流。
