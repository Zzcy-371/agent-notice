# Agent 技术日报 · 2026-07-13

## GitHub projects

### [superpowers](https://github.com/obra/superpowers)
- 集成MCP（模型上下文协议）实现标准化的工具调用与资源访问，提升Agent开发效率。
- 结合RAG（检索增强生成）技术，增强模型对知识库的检索与推理能力。
- 支持多智能体（multi-agent）协作架构，通过tool calling机制实现复杂任务分解与执行。

### [ECC 代理框架性能优化系统](https://github.com/affaan-m/ECC)
- 集成MCP协议与RAG记忆系统提升工具调用上下文一致性
- 支持Claude Code、Codex、Opencode等IDE的多代理协作与调度优化
- 执行tool calling时自动应用安全性约束与技能惯性学习机制

### [hermes-agent](https://github.com/NousResearch/hermes-agent)
- 支持 MCP（模型上下文协议）集成，可实现工具调用与外部数据源的无缝交互。
- 内置 RAG 检索增强生成能力，使知识库动态扩展，回答更精准。
- 采用多智能体架构，各 agent 协同配合，支持复杂任务分解与并行处理。

### [claw-code](https://github.com/ultraworkers/claw-code)
- 通过 MCP 协议与 Rust 后端交换具有自我解释能力的符号化提示块（Gajae-Code / LazyCodex），实现基于 RAG 的动态展品叙事生成。
- 调用 tool calling 与链上数据库交互，自动检索并验证展品元数据，确保所有内容片段符合博物馆策展规则。
- 以多智能体架构管理展区行为：每个智能体独立负责灯光、音效或文字解说，通过共享状态协调无人类干预的运行闭环。

### [opencode](https://github.com/anomalyco/opencode)
- MCP：opencode 通过 MCP 协议与 Claude 等模型交互，实现代码生成与执行。
- RAG：结合 RAG 技术检索代码库与文档，提升上下文理解能力。
- Tool calling & Multi-agent：支持工具调用和环境操作，并协调多智能体处理复杂任务。

### [skills](https://github.com/anthropics/skills)
- 整合MCP协议与RAG检索增强能力，实现Agent技能模块的标准化远程调用。
- 支持tool calling动态编排，通过多智能体(multi-agent)协作扩展技能组合的灵活性与复杂度。
- 提供技能仓库的公开模板与最佳实践，便于社区贡献与复用Agent技能组件。
